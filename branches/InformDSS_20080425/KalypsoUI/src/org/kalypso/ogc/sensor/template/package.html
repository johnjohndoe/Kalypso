Die Zeitzone in der Diagramm- und Tabellendarstellung kann durch die Vorlage definiert sein. Dazu habe ich, wie telefonisch mit euch gesprochen, die Vorlagen um den optionalen "timezone"-Element erweitert. Ist es vorhanden, dann wird die passende Zeitzone genommen und für die Darstellung benutzt. Ist es nicht vorhanden, dann wird die Default-Timezone genommen (Locale). Beim speichern einer Vorlage wird die Zeitzone nur dann geschrieben, wenn sie sich von der Default-TZ differenziert. Der Wert vom "timezone"-Element ist ein String und ist gleich der ID wie in java.util.TimeZone.getTimeZone( String ID ). Beipsiel: - Europe/Berlin - America/Los_Angeles - GMT+1:00 - GMT-2:00 wobei die zwei letzten "deprecated" sind. Man soll ein Fullname benutzen wie Europe/Berlin. Mehr Infos über Zeitzonen: http://www.timeanddate.com/ NB: es gibt z.Z. keine GUI (z.B. Dialog) um die Zeitzone zu ändern, man muss es in der Vorlage (also .ott oder .odt) per Hand machen.
