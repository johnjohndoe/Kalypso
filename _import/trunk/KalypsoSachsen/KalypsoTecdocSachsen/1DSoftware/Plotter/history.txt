plotter-exe-2-a-0-0 30.8.2002
-------------------
- Anpassung an Änderungen in commonMfc.lib
- WspPrj.dll in statische Lib umgewandelt
- Der Benutzer kann jetzt die Ausrichtung auf dem Papier von Profil und Tabelle festlegen.
- Der Stempel kann jetzt neuerdings auch am Profil ausgerichtet werden ( früher stets nur am gegenüberliegenden Seitenrand ).
- Bugfix: wird nur ein Teil des Profils angezeigt ( linke/rechte
Begrenzung ), wird die Höhe entsprechend adaptiert
- Bugfix: Objekte können nicht mehr ausserhalb des Anzeigebereichs
verschoben werden
- CDrawObj: neues Flag 'offsetable': solche Objekte können zwar
verschoben werden, es ändert sich dann aber nur ihr offset
- Titel, Kommentar und Bezugshöhe sind 'offsetable'
- Performance: InvallAll wird nur noch einmal im
CPlotterDoc::UpdateDrawing aufgerufen
- Nach Anwendung eines neuen Templates waren bei Wsp-Höhen stets die untere Bezeichnung gleich der oberen; gefixt
- Wsp-Höhen wurden oft nur als ein Polygonzug dargestellt, unterbrochene haben nicht funktioniert
- Attribut Hideable für DrawObjekte implementiert: die meisten Graphikelemente lassen sich jetzt einfach verstecken ( Profilelemente, Koordinaten, Stempel )
- Template an hideable angepasst
- Datenblockart Buhnen wird nicht mehr unterstützt
- DXF - Export: Texte stehen jetzt an richtiger Stelle ( bis auf Ausrichtung )
- User Objekte können jetzt am Profil fixiert werden ( neuer Menupunkt )
- Objekte können nicht mehr ( weit ) aus dem Zeichenbereich verschoben werden
- Ausrechnen der Profilausdehnung neu programmiert ( nötig wegen profilfixierter Userobjekte )
- Versionsnummer wird jetzt aus der Exe ausgelesen
- Context-Sensitive Hilfe: benutzt jetzt CommonMfc::CContextHelp und ruft plotter.pdf auf
- Skalieren des Stempels möglich -> Eigenschaften-Ausrichtung
- Es werden jetzt die Templates Rect und point aus der commonMfc benutzt, dadurch sind die benutzten Rechtecke
stets richtig orientiert
- Bugfix: Polygonobjekte haben nicht mehr die Einstellmöglichkeiten für Pfeilspitzen
- neue Moglichkeit ( bislang nur Intern ) die Schrift eines Objekts zu skalieren. Wird beim Stempelskalieren benutzt.
- Bugfix: die Anzahl der Nachkommastellen wurde nicht aus den Vorlagen übernommen	
- Es können jetzt ( potentiell ) beliebig viele Zeilen pro Legendeneintrag dargestellt werden. Konkret wird bei den 
Datensätzen WSP-HOHEE, Deiche, Ausuferung etc. die zweite Zeile des Datenbocks, falls nicht leer, als dritte
Zeile in der Legende dargestellt ( Extra für Stuchly, damit die Qs in der WSP-Hoehe angegeben werden )
- neue Möglichkeit zur Einschränkung des Profilsbereichs: die Darstellung des Profils kann Anhand eines
Datensatzes eingeschränkt werden ( alternativ zur Angabe 'automatisch' bzw. eigener Werte ). Wird z.B. auf den
DatenBlock 'Trennflächen' eingeschränkt, so wird nur alles innerhalb der Trennflächen dargestellt
- Fuktion GetImageType für Datenblöcke aus CPlotterApp in die WspPrj.lib::DataBlock verschoben
