Änderungen im Schema,

Fall 1:
	optionales Element kommt hinzu
	Bedingungen:
	- bestehende GML-Daten bleiben valid
	- Server und Client können alle mit dem neuen Schema arbeiten
	- GML-daten müssen nicht angepasst werden.
	-> NS bleibt gleich, SchemaVersion++, Alle Cataloge (Server und Client) benutzen neuste Version

Fall 2:
	Element fällt weg
	Bedingungen:
	- Niemand braucht das Element
	-> wie Fall 1.

Fall 3:
	komplexere Änderungen
	-> neuer NS (unterschiedliche Endung im NS-namen z.B. xyz_V1.2.3)




Ideen:
ConverterRegistry bietet methoden zum convertieren an.
(Beide Richtungen)

GUI: Dialog auf Projekt oder Verzeichnis
Variante:
 A converter entscheidet ob er convertieren kann.
 B registry entscheidet welche converter gültig sind
