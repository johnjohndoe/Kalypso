Managing Project Tree


Project Tree visualise data hierarchy. Top level consists of server on which client connects on DB.
For structuring all thematic data sets of RRM following elements are used:
Project
Theme
Stady
Version



Edit Structure Elements

Context Menu for versions:

REMOVE enables to drop structure elements.

RENAME enables to name versions new also versions can be moved to other projects or for ModelData to other states.

COPY enables to create copy of selected version to save existing version. Remember that every change is directly stored in database.

Via XML-IMPORT in context menu of active version user can add data of existing versions to new created version. For Control and SimCase theme new versions can be built to setup simulation. This functionality also enables the user to export a ModelData version from DB to file system.
This is needed for the Kalypso Forecast Tool because this program application bases on a calibrated and validated ModelData version. If a Modeldata version is imported to an existing version all data sets will be overwritten.
